# Repository Guidelines

## Project Structure & Module Organization
This repository is a small Python utility centered on [`currency.py`](/Users/gordon/codex/currency.py), which handles CLI parsing, API access, cache management, and table rendering. Runtime configuration lives in `.env` and should provide `CURRENCY_API_KEY` and optional `CURRENCIES`. Cached API metadata is stored under `.currency_converter/supported_currencies.json`. `__pycache__/` contains generated bytecode and should not be edited manually.

## Build, Test, and Development Commands
Use the project virtualenv interpreter directly:

- `/Users/gordon/python/currency/bin/python3 -m py_compile currency.py` checks syntax.
- `./currency.py --help` shows CLI options.
- `./currency.py -c USD -a 10` runs a non-interactive conversion against the configured target currencies.
- `./currency.py` starts the interactive prompt mode.
- `/Users/gordon/python/currency/bin/pip install -r requirements.txt` is not available here yet; install dependencies individually if needed, for example `pip install tabulate requests python-dotenv`.

## Coding Style & Naming Conventions
Follow standard Python style: 4-space indentation, `snake_case` for functions and variables, and short focused helpers. Keep user-facing error messages explicit and preserve existing fallback behavior for cache and network failures. Prefer adding logic to small functions rather than expanding `main()` further.

## Testing Guidelines
There is no formal test suite yet. For changes, at minimum run `py_compile` and exercise both CLI paths:

- `./currency.py -c USD -a 10`
- `./currency.py` and enter a valid currency code

If you add tests, use `pytest`, place them in `tests/`, and name files `test_*.py`.

## Commit & Pull Request Guidelines
This directory is not currently a Git repository, so no local commit history is available to infer conventions. Use short imperative commit messages such as `Add tabulated CLI output` or `Preserve cache metadata fallback`. For pull requests, include a brief summary, note any config or dependency changes, and paste example terminal output when the table format changes.

## Security & Configuration Tips
Do not commit real API keys. Keep secrets in `.env` or a home-directory `.env`, and verify network-dependent changes with a valid `freecurrencyapi` key. Treat `.currency_converter/` as generated state; it is safe to refresh when validating cache behavior.
